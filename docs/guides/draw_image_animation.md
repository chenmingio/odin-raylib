Draw Image/Animate设计

原理

画一个静态图像

- 确定Buffer上的minX/Y和maxX/Y。这个通常是由不同的情况共同决定的
- Clip以后，source上开始的地方也要移动（可以理解为原图像被遮挡了）
- 

画一个动态图像
- 需要记住画到第几帧了，下一次要“续播”
- 

使用AstFrame画动态图像
- Source起始的位置需要由row/frame/offset共同决定
- frameIndex，压缩，

简化版本：

先用intersect(buffer, sprint)求出buffer上需要作画的区域，然后loop这个区域（同时反推出source上的value）。这样不需要clip了。

  1. 静态图像 Clip

  你的 draw_image_corp 里有两层偏移：

  // render.odin:140-147

  // 第一层：屏幕裁剪（buffer 边界）
  minX := clamp(pos.x, 0, buffer.width)
  maxX := clamp(pos.x + size.x, minX, buffer.width)

  // 第二层：source 偏移 = 屏幕裁剪偏移 + 图集偏移
  offset_x := (pos.x >= 0 ? 0 : -pos.x) + offset.x
  //          ↑ 左边被裁掉多少      ↑ 图集中的起始位置

  屏幕裁剪示意：
  ┌─────────────────┐ buffer
  │     ┌───────────┼──┐ 图像
  │     │ 可见区域  │  │
  │     └───────────┼──┘
  └─────────────────┘
        ↑           ↑
       minX        maxX（被 buffer.width 裁掉）

  2. 动画帧切换

  // render.odin:204-208

  entity.anim_time += i32(dt * 1000)  // 累积时间（毫秒）

  // 如果累积时间超过当前帧时长，切换到下一帧
  for entity.anim_time >= clips[entity.anim_frame_idx].duration {
      entity.anim_time -= clips[entity.anim_frame_idx].duration
      entity.anim_frame_idx = (entity.anim_frame_idx + 1) % i32(len(clips))
  }

  用 for 而不是 if，是因为掉帧时可能需要跳过多帧。

  你可能下一步会遇到的

  1. Z-order 排序

  2D 游戏里，Y 值小的（屏幕上方）应该先画，被 Y 值大的遮挡：

  // 按 Y 排序后再绘制
  slice.sort_by(entities, proc(a, b: Entity) -> bool {
      return a.pos.offset.y > b.pos.offset.y
  })

  2. 动画事件

  攻击动画第 3 帧触发伤害判定：

  AnimEvent :: struct {
      frame_idx: int,
      event_type: enum { HitBox, Sound, Particle },
  }

  // 检测帧切换时触发事件
  if old_frame != new_frame {
      for event in clip.events {
          if event.frame_idx == new_frame {
              trigger(event)
          }
      }
  }

  3. 动画过渡

  Idle → Run：可以硬切
  Run → Idle：最好等当前帧播完，或者做 1-2 帧的过渡
  Attack → Idle：必须等攻击动画播完

  需要区分 可打断 和 不可打断 动画。

  4. 九宫格（9-slice）

  UI 元素（按钮、面板）拉伸时，边角不变形：

  ┌───┬─────┬───┐
  │ 1 │  2  │ 3 │  ← 角落固定
  ├───┼─────┼───┤
  │ 4 │  5  │ 6 │  ← 边缘只拉伸一个方向
  ├───┼─────┼───┤
  │ 7 │  8  │ 9 │  ← 中心双向拉伸
  └───┴─────┴───┘
