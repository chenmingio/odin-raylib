# 碰撞系统

**目的**：检测entity之间的碰撞，计算碰撞时间和响应
**适用于**：sim region内的entity模拟

---

## 1. 化约问题

两个移动entity的碰撞 → 参考系变换 → 一个静止 + 一个移动（相对速度）

### Minkowski和

- 朴素理解：两个方框的碰撞，等价为一个点 + 一个四边形（两个方框尺寸叠加）的碰撞
- 理论形式：
    ```
    A ∩ B ≠ ∅  ⟺  0 ∈ (A ⊕ (-B))
    ```
  - 两个集合的Minkowski sum定义：`A ⊕ B = {a+b | a∈A, b∈B}`
  - 判断两个图形是否相交，等价于判断0是否在Minkowski差里
- 证明：需要正反两方向证明"等价于"
  - 正向：若A∩B非空，则存在公共点p，p + (-p) = 0，所以0在A⊕(-B)里
  - 反向：若0在A⊕(-B)里，则存在a∈A, b∈B使a-b=0，即a=b，所以A∩B非空
- 几何直觉：
  - 理论没有规定原点位置
  - 把A缩成中心点，把B扩大A的尺寸
  - A的中心点在扩大后的B内部 ⟺ 碰撞
  - 从1维到2维：
    - 1维：两个线段重叠 ⟺ 一个线段的中心在 (B ⊕ A) 内
    - 2维：两个矩形碰撞 ⟺ 一个矩形的中心在 (B ⊕ A) 内

### GJK算法（通用凸形状）

- 不直接计算整个Minkowski和（O(n²)太慢）
- 用support function迭代逼近（O(k·n)）
- 适用于任意凸多边形，本项目用AABB不需要

## 2. 碰撞时间计算

化约后的问题：点 + 速度向量 vs 矩形，求碰撞时间 t ∈ [0, 1]

### 区间交集法

- 降维思路：2维问题拆成两个1维问题
- X方向：射线到达左/右边的时间 → 得到区间 [t_enter_x, t_exit_x]
- Y方向：射线到达上/下边的时间 → 得到区间 [t_enter_y, t_exit_y]
- 碰撞时间 = 两个区间的交集
  - `t_enter = max(t_enter_x, t_enter_y)` ← 两轴都进入才算碰撞
  - `t_exit  = min(t_exit_x, t_exit_y)`   ← 任一轴离开就算离开
  - `t_enter ≤ t_exit` → 碰撞，否则错过
- 特殊情况：
  - 速度平行于某轴 → 该轴t无定义，检查起点是否在范围内
  - t_enter < 0 → 起点已在矩形内部

### 逐边检验法（Handmade Hero）

- 对矩形每条边，计算射线到达的时间t
- 排除无效的t：
  - t < 0（在背后）
  - t > 1（超出本帧）
  - 到达时另一轴不在边的范围内（碰到延长线，不是边本身）
- 留下最小的t = 碰撞时间
- 优势：自然得到碰撞法线（是哪条边产生的碰撞）

## 3. 碰撞响应

检测到碰撞后，用法线决定反应：

| 类型 | 公式 | 用途 |
|------|------|------|
| 滑动 | `v -= n * dot(v, n)` | 角色撞墙 |
| 反弹 | `v -= 2 * n * dot(v, n)` | 弹球 |
| 停止 | `v = 0` | 简单实现 |

## 4. 多entity碰撞

### 问题

计算顺序影响结果：B和C先碰撞，但计算从A开始，处理A-B碰撞后B位置变了，B-C的碰撞结果失效。

### 解决：迭代求解

1. 找到整个帧内**最早**的碰撞
2. 所有物体移动到该时刻
3. 处理该碰撞
4. 用剩余时间重复步骤1
5. 最多迭代4次（防止死循环）

---

## 5. 碰撞体积

- 基本形状：AABB（轴对齐矩形）
- 复合形状：多个AABB组合
- 本项目暂时只用单个AABB

---

## 6. 代码流程

1. 用Minkowski和化约为：A的中心点 vs 扩大后的B（B ⊕ A的形状）
2. 用逐边检验法计算碰撞时间t和法线n
3. 移动到碰撞点，用法线计算响应（滑动）
4. 用剩余时间迭代检测下一次碰撞
