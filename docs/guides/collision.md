# 碰撞系统

**目的**：检测entity之间的碰撞，计算碰撞时间和响应
**适用于**：sim region内的entity模拟

---

## 1. 化约问题

两个移动entity的碰撞 → 参考系变换 → 一个静止 + 一个移动（相对速度）

### Minkowski和

- 朴素理解：两个方框的碰撞，等价为一个点 + 一个四边形（两个方框尺寸叠加）的碰撞
- 理论形式：
    ```
    A ∩ B ≠ ∅  ⟺  0 ∈ (A ⊕ (-B))
    ```
  - 两个集合的Minkowski sum定义：`A ⊕ B = {a+b | a∈A, b∈B}`
  - 判断两个图形是否相交，等价于判断0是否在Minkowski差里
- 证明：需要正反两方向证明"等价于"
  - 正向：若A∩B非空，则存在公共点p，p + (-p) = 0，所以0在A⊕(-B)里
  - 反向：若0在A⊕(-B)里，则存在a∈A, b∈B使a-b=0，即a=b，所以A∩B非空
- 几何直觉：
  - 理论没有规定原点位置
  - 把A缩成中心点，把B扩大A的尺寸
  - A的中心点在扩大后的B内部 ⟺ 碰撞
  - 从1维到2维：
    - 1维：两个线段重叠 ⟺ 一个线段的中心在 (B ⊕ A) 内
    - 2维：两个矩形碰撞 ⟺ 一个矩形的中心在 (B ⊕ A) 内

### GJK算法（通用凸形状）

- 不直接计算整个Minkowski和（O(n²)太慢）
- 用support function迭代逼近（O(k·n)）
- 适用于任意凸多边形，本项目用AABB不需要

## 2. 碰撞时间计算

化约后的问题：点 + 速度向量 vs 矩形，求碰撞时间 t ∈ [0, 1]

### 区间交集法

- 降维思路：2维问题拆成两个1维问题
- X方向：射线到达左/右边的时间 → 得到区间 [t_enter_x, t_exit_x]
- Y方向：射线到达上/下边的时间 → 得到区间 [t_enter_y, t_exit_y]
- 碰撞时间 = 两个区间的交集
  - `t_enter = max(t_enter_x, t_enter_y)` ← 两轴都进入才算碰撞
  - `t_exit  = min(t_exit_x, t_exit_y)`   ← 任一轴离开就算离开
  - `t_enter ≤ t_exit` → 碰撞，否则错过
- 特殊情况：
  - 速度平行于某轴 → 该轴t无定义，检查起点是否在范围内
  - t_enter < 0 → 起点已在矩形内部

### 逐边检验法（Handmade Hero）

- 对矩形每条边，计算射线到达的时间t
- 排除无效的t：
  - t < 0（在背后）
  - t > 1（超出本帧）
  - 到达时另一轴不在边的范围内（碰到延长线，不是边本身）
- 留下最小的t = 碰撞时间
- 优势：自然得到碰撞法线（是哪条边产生的碰撞）

## 3. 碰撞响应

检测到碰撞后，用法线决定反应：

| 类型 | 公式 | 用途 |
|------|------|------|
| 滑动 | `v -= n * dot(v, n)` | 角色撞墙 |
| 反弹 | `v -= 2 * n * dot(v, n)` | 弹球 |
| 停止 | `v = 0` | 简单实现 |

## 4. 接触、浮点误差与初始重叠

### 保持 sweep fraction 精确

- 对当前位移线段 `p(s) = p0 + dp_remaining * s`，sweep 求出的 `t_enter` / `sweep_fraction` 是首次接触的参数 `s ∈ [0, 1]`，即该线段已走距离的比例。它也可称为 normalized time of impact（TOI），但在本项目中不表示物理时间。
- 命中后应移动 `dp_remaining * sweep_fraction`，并以 `dp_remaining * (1 - sweep_fraction)` 作为剩余位移；不要为了留缝而修改该比例，例如 `sweep_fraction - 0.001`。
- `sweep_fraction - 0.001` 的 `0.001` 是比例，不是世界距离：实际法线回退量约为 `length(dp_remaining) * 0.001`。速度更快或帧更长时，留缝会更大；慢速时则更小。
- 该比例回退还会把剩余段增大 `dp_remaining * 0.001`。碰撞响应保留切线分量后，角色会额外滑动相应的切线距离；这段位移不属于真实命中点之后的运动。

### 贴边时允许退出

角色位于接触面时，只应阻止它继续向墙内运动；沿墙切向移动、或朝墙外移动都必须允许。这样即使浮点舍入让角色略微进入墙内，也不会被卡住。

- 对某个面的运动方向指向外侧：不将该面视为本次阻挡命中。
- 对某个面的运动方向指向内侧：移除速度和剩余位移的法线向内分量。
- 因而，在“精确接触 + 去掉法线向内分量”的正常路径中，不需要为了防止粘住而回退或添加 skin。

### 初始重叠

sweep AABB 解决的是“从不重叠到首次接触”，不是通用的穿透修正。若起点已经在 Minkowski AABB 内（`t_enter < 0`），应单独处理：

- 仅有极小重叠（`overlap_slop` 范围内）且位移朝最近出口方向：允许本帧退出，不报阻挡 hit。
- 位移继续朝内，或重叠明显大于 `overlap_slop`：按最小穿透方向将物体推出，或至少移除朝内的速度分量。
- 即使允许从当前墙体退出，仍要对移动路径检测其他墙体。

### skin 的取舍

`skin` 是在 Minkowski AABB 外再扩出的很小安全边距；它让角色在真实边界外提前接触，抵消浮点误差。它不是处理粘住的必要条件，并会使“只进入 skin、未进入真实碰撞体”的状态需要额外分类。

若实际观察到精确 TOI 仍偶发产生穿透，可选择：

1. 保持精确 `sweep_fraction`，移动到接触点后仅沿法线向墙外做很小的固定世界距离位置修正；或
2. 使用 skin，并将其当作接触容差，而非真实穿透。

固定距离修正的效果是：留缝恒为 `skin`，且剩余位移仍按真实的 `1 - sweep_fraction` 计算。两种方案都应将安全距离与 sweep fraction 分离，不用 `sweep_fraction - epsilon` 代替两者。

### 角点

若 X、Y 两轴的进入时间近似相等，射线命中的是角点。此时需要同时处理两条法线（移除两个朝内分量）；只选其中一条边会让剩余位移进入另一条边，造成偶发穿边。

## 5. 多entity碰撞

### 问题

计算顺序影响结果：B和C先碰撞，但计算从A开始，处理A-B碰撞后B位置变了，B-C的碰撞结果失效。

### 解决：迭代求解

1. 找到整个帧内**最早**的碰撞
2. 所有物体移动到该时刻
3. 处理该碰撞
4. 用剩余时间重复步骤1
5. 最多迭代4次（防止死循环）

## 6. 碰撞体积

- 基本形状：AABB（轴对齐矩形）
- 复合形状：多个AABB组合
- 本项目暂时只用单个AABB

---

## 7. 代码流程

1. 用Minkowski和化约为：A的中心点 vs 扩大后的B（B ⊕ A的形状）
2. 用逐边检验法或区间交集法计算精确 `sweep_fraction` 和法线
3. 移动到精确碰撞点；仅移除法线向内的运动，保留离墙和切向运动
4. 用剩余时间迭代检测下一次碰撞
